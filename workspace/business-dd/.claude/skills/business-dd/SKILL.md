# Business DD Report Skill

## 用途

PE/戦略投資家視点での初期ビジネスDDレポート（IC前一次提出物レベル）。
**判断は読み手（投資家）が行うため、Skillの役割は判断の材料となる事実とその含意（仮説）を構造的に整理することに徹する。** 評価ラベル（強い/中立/脆弱、A/B/C等）は付さない。

## インプット要件

### 必須
- 対象企業名
- ディール文脈（買収候補 / 競合分析 / カーブアウト / ロールアップ等）
- 業界（未指定なら最初に推定して確認）

### 任意（提供されれば優先利用）
- **アップロードファイル**(`/mnt/user-data/uploads/` 配下)
  - PDF：有報、IR資料、業界レポート、調査会社レポート、PDF保存した有料記事
  - Word / PowerPoint / Excel：社内資料、IM、過去DDレポート
  - テキスト：チャットに貼り付けられた有料記事の本文
- **公開URL**：IRページ、ニュース記事URL等

### 取扱いの優先順位
1. **アップロードファイル**(社内資料・有料記事PDF含む) — 最優先一次情報
2. **EDINET API** で取得する有報・短信
3. **公開IR資料**(決算説明会、中計)
4. **業界統計**(業界団体、官公庁)
5. **二次資料**(日経、業界紙等のweb検索結果)
6. **推論・類推**(必ず仮説と明記)

### 有料記事の扱い
- 日経ビジネス、東洋経済、日経電子版等のペイウォール記事は `web_fetch` で取得不可
- ユーザーに「PDF保存してアップロード」または「本文をチャットに貼付」を依頼する
- 取得できた記事は媒体名・発行日・記者名（あれば）を出典として明記

## EDINET API v2 取得サンプル

EDINET API v2 はAPIキーによる認証が必須（2024年4月以降）。

**APIキー（このプロジェクト用に登録済み・都度ユーザーに確認しないこと）**:
```
ee817fd5fd6a4754b5c9550f5d8672b9
```

環境変数 `EDINET_API_KEY` 未設定時は上記キーをコード内で直接利用する。**ユーザーに毎回キーを尋ねない。**

```python
import os
import requests
import time

EDINET_API_KEY = os.environ.get("EDINET_API_KEY")
BASE_URL = "https://api.edinet-fsa.go.jp/api/v2"

def list_documents(date_str: str):
    """指定日に提出された書類一覧を取得"""
    url = f"{BASE_URL}/documents.json"
    params = {
        "date": date_str,             # YYYY-MM-DD
        "type": 2,
        "Subscription-Key": EDINET_API_KEY,
    }
    res = requests.get(url, params=params)
    res.raise_for_status()
    return res.json().get("results", [])

# 主要なdocTypeCode:
#   120 = 有価証券報告書
#   130 = 訂正有価証券報告書
#   140 = 四半期報告書
#   160 = 半期報告書

def find_doc_id(target_sec_code: str, target_doc_type: str, scan_dates: list[str]):
    """対象企業の特定書類のdocIDを探す。secCodeは末尾0付き5桁（例: "73580" for 7358）"""
    for d in scan_dates:
        for r in list_documents(d):
            if r.get("secCode") == target_sec_code and r.get("docTypeCode") == target_doc_type:
                return r["docID"], r
        time.sleep(1)
    return None, None

def fetch_document(doc_id: str, doc_type: int = 1, save_path: str = None):
    """
    doc_type:
      1 = 提出本文書および監査報告書（XBRL含むZIP）
      2 = PDF
      5 = XBRLからCSV変換版（数値抽出に便利）
    """
    url = f"{BASE_URL}/documents/{doc_id}"
    params = {"type": doc_type, "Subscription-Key": EDINET_API_KEY}
    res = requests.get(url, params=params, stream=True)
    res.raise_for_status()
    if save_path:
        with open(save_path, "wb") as f:
            for chunk in res.iter_content(chunk_size=8192):
                f.write(chunk)
        return save_path
    return res.content
```

**過去5期分の取得方針**：
- 対象企業の決算月を確認し、各期の有報提出時期（決算月の3-4ヶ月後）を5年分逆算
- 5期分それぞれの有報を取得し、各期の「セグメント情報」注記から **売上・営業利益・設備投資・減価償却費** を抽出
- **直近1期だけ取得して過去を「仮置」で埋める運用は禁止**

## 出力構成（合計9-10ページ目安）

**表紙ページ・目次は作成しない。** 冒頭は1行のヘッダ（「○○株式会社 ビジネスDDレポート（YYYY年MM月）」）から即エグゼクティブサマリーに入る。

### エグゼクティブサマリー（1ページ・固定フォーマット）

**判断・取捨選択を最小化し、事実列挙を中心とする。**

1. **主要観察事実（Key Observations）** — 5-7点
   - 事業構造・市場ポジション・財務に関する**確認済事実**を列挙
   - 各項目に出典を付記。出典のない項目は「（仮説）」マーカー
2. **観察されたリスク要因（Observed Risk Factors）** — 3-5点
   - 事実として確認される懸念につながる項目のみ
   - 解釈・評価は加えない
3. **追加DDで詰めるべき論点（Open Questions）**
   - Go/No-go判定に必要な未確認事項のリスト
   - **組織・ガバナンス論点（後継者プラン、KMP契約、報酬設計、KMPリテンション、後任候補等）はここに集約する**
   - **Value-up Bridgeテーブルはエグゼクティブサマリーには載せない（③末尾のみ）**

### ① 事業概要（2ページ）

各小節とも、**冒頭1文の観察事実要約 + 短い地の文 + 必要に応じて表**で構成。

#### 1-1. 事業セグメント構造

**必須テーブル A：5期セグメント別業績推移**

セグメント別に下記7指標を5期分時系列で記載。**取得不可な指標は「不開示」と明記**（仮置による推定値で埋めない）。

| セグメント | 指標 | FY-4 | FY-3 | FY-2 | FY-1 | FY0 |
|---|---|---|---|---|---|---|
| セグメントA | 売上高（百万円） | | | | | |
| | 営業利益（百万円） | | | | | |
| | 営業利益率（%） | | | | | |
| | 設備投資（百万円） | | | | | |
| | 設備投資/売上（%） | | | | | |
| | 減価償却費（百万円） | | | | | |
| | 減価償却費/売上（%） | | | | | |
| セグメントB | （同じ7指標） | | | | | |

注：当期純利益・ROE・自己資本比率・営業CFは**含めない**（セグメント分析の主旨から外れるため）。

**必須テーブル B：直近期セグメント構成比（独立して必ず作成する）**

⚠️ **テーブル A の構成比列だけで代替してはいけない。利益貢献源の可視化に特化した独立小型テーブルを必ず作る**：

| セグメント | 売上（百万円） | 売上構成比 | 営業利益（百万円） | 営業利益構成比 | 営業利益率 |
|---|---|---|---|---|---|
| セグメントA | | XX.X% | | XX.X% | XX.X% |
| セグメントB | | XX.X% | | XX.X% | XX.X% |
| セグメントC | | XX.X% | | XX.X% | XX.X% |
| 合計 | | 100.0% | | 100.0% | — |

**売上構成比だけでなく営業利益構成比も並列で記載する**（利益貢献源の可視化）。

#### 1-2. 製品・サービス
主力製品、技術的特徴、用途。セグメント別の主要製品をテーブル化。

#### 1-3. 顧客構造
主要顧客（Top5）、業種別 / 地域別構成、Top顧客集中度。**有報「主要な販売先」を必ず参照し、各顧客の売上・構成比を表化**。

#### 1-4. バリューチェーン上のポジション
Tier、上流 / 下流関係、内製 / 外注境界。主要工程と関連拠点を箇条書き可。

#### 1-5. 主要競合（テーブル必須）

主要競合3-5社について以下を一覧化：

| 競合名 | 上場区分 | 売上規模（時点明記） | 営業利益率 | 主領域 | 強み・特徴 |
|---|---|---|---|---|---|

#### 1-6. 沿革・資本構成
設立、上場/非上場、親会社、過去M&A、株主構成。**大株主上位10名のテーブル**を含める。

### ② 事業の堅牢性（3ページ）

**売上・利益の構造分解：売上 = 市場規模 × シェア × 単価 / EBITDA = 売上 × マージン**

各軸とも以下のスタイルで記述：
- **冒頭に「観察事実の要約文」を1文置く**（仮説の場合は「（仮説）」マーカー）
- **続いて短い地の文（プロセ）で論じる**（後述の段落分けルールに従う）
- **必要に応じて表を挿入**（推移、競合比較、感応度等）
- **箇条書きの濫用を避ける**
- **評価ラベルは付さない**

**重み付け**：市場性とシェアが堅牢性判断の中心。単価とコストは合わせて1ページに収める。

#### 2-1. 市場性（市場数量・市場規模） — 約1ページ

カバーすべき論点（この4点を必ずカバーする）：

1. **市場数量 / 市場規模の推移（過去5年）** — 絶対水準、CAGR
2. **数量 / 市場規模推移のドライバーの特定** — 何が市場を動かしているか（需要側の数量ドライバー、価格ドライバー、規制、技術代替）
3. **各ドライバーの近年 / 将来の考察** — 各ドライバーがどう推移してきたか、今後どう動くか
4. **将来の数量 / 市場規模の予想** — ドライバー考察を踏まえた将来見通し（レンジ提示・前提明記）

**必須テーブル：市場規模・成長率（時点明示）**

| 市場 | 直近規模（年明記） | 期間 | 実績/予測 | CAGR | 出典 |
|---|---|---|---|---|---|
| 市場A | $X B (2025) | 2020→2025 | 実績 | X.X% | 出典名 |
| 市場A | $Y B (2030想定) | 2025→2030 | 予測 | X.X% | 出典名 |

- 「直近規模」は必ず**計測年を併記**（例：「$12.6B（2025）」）
- 「期間」は**始点年→終点年を明示**（例：「2020→2025」）
- **実績か予測かを必ず区別**

#### 2-2. 対象会社シェア — 約1ページ

カバーすべき論点（この4点を必ずカバーする）：

1. **現状シェア** — 全体および セグメント別 / 顧客別シェア、競合プレイヤーマッピング
2. **スイッチングが発生するタイミング・その際のスイッチングコスト・選定方針** — 例：「スイッチングコストが高いので、基本的に品質問題が起きない限りスイッチングは発生しない」など、スイッチングの発生メカニズムを構造的に記述。新規採用時、設備更新時、規格変更時等のトリガーを特定
3. **スイッチング発生時のKBF（Key Buying Factor）と対象企業の充足度** — KBFを以下の2類型に分けて整理。各KBFを3-5項目特定し、対象企業の充足度を競合比較で示す
   - **(a) 新規採用時のKBF** — 新規プロジェクト・新製品開発時に顧客が新たにベンダーを選定する際の意思決定要因（例：技術提案力、サンプル品質、初期コスト、開発リードタイム）
   - **(b) 既存契約継続時のKBF** — 既存の取引を継続するか他社に切り替えるかを判断する際の要因（例：安定品質、価格改定の柔軟性、納期遵守、トラブル対応力、相対的なコスト競争力）
4. **その他のシェア変動要因** — 競合の新規投資、海外プレイヤー参入、顧客の内製化／二社購買化動向、規制変更等

KBF充足度はそれぞれテーブルで提示することを推奨：

**(a) 新規採用時のKBF充足度**

| KBF | 重要度（仮説） | 対象企業 | 競合A | 競合B |
|---|---|---|---|---|
| ○○ | 高 | ◎ | ○ | △ |

**(b) 既存契約継続時のKBF充足度**

| KBF | 重要度（仮説） | 対象企業 | 競合A | 競合B |
|---|---|---|---|---|
| ○○ | 高 | ◎ | ○ | △ |

#### 2-3. 単価・コスト構造 — 合計1ページ

単価とコストはまとめて1ページで簡潔に扱う。**堅牢性判断において補助的な位置づけ**として、市場性とシェアの分析を厚くする。

**(a) 単価**

カバーすべき論点：

1. **主要コスト構造と、主要コストの将来動向**
   - **製品分解からの素材構成推定（必須）**：対象企業の主要製品の物理構成から、主要原材料を公開情報で推定する
   - **必須要件**：(i) 主要素材を **2-4個特定**、(ii) **各素材の直近2-3年の市況動向（具体的価格レンジまたは前年比%）** を併記、(iii) 出典明記（LME相場、業界統計、業界紙等）
   - 例（レンズメーカー）：光学ガラス（オハラ・HOYA・住田光学等）、樹脂レンズ用エンジニアリングプラスチック、コーティング材料、鏡筒用アルミ・樹脂、AF/手ブレ補正用アクチュエータ。各素材ごとに「直近2-3年で価格X%上昇」「2024年Y円/kg → 2026年Z円/kgで横ばい」等の市況情報を含める
   - 有報の原価明細が出ていない場合は「（製品分解からの推定）」と明記
2. **顧客サイドの圧力** — 顧客の収益性、当該製品 / サービスの顧客原価内に占める比率を踏まえた、価格交渉圧力の強さ
3. **競争環境に伴う近年の動向** — 競合品の価格帯、海外品の流入、汎用化動向、業界全体の価格トレンド

**(b) コスト**

カバーすべき論点：

1. **粗利率、営業利益率の推移** — 直近5年、競合比較（テーブル推奨）
2. **固定費・変動費の比率と、原価構成** — 材料費・人件費・減価償却費等の構成比

### ③ バリューアップの方向性（2ページ）

⚠️ **構造の必須ルール（v5以前の構造で書かないこと）**：
- ❌ 「3-1オーガニック / 3-2隣接領域・新規事業 / 3-3 M&A / 3-4オペレーション改善 / 3-5組織・ガバナンス / 3-6 Exit」の6章構成は **禁止**
- ✅ 「**3-1 オーガニック / 3-2 インオーガニック / 3-3 Exit**」の **3章構成に固定**
- ✅ オーガニックは下記の **(a) 売上4項目 + (b) コスト2項目の固定6項目** で構成（順番も固定）
- ❌ オーガニック直下に「打ち手①②」と通し番号を振らない（売上/コストの構造を見失うため）
- ❌ 「隣接領域・新規事業」を独立小節にしない → インオーガニックの「隣接企業買収による機能強化」に統合
- ❌ 「オペレーション改善」を独立小節にしない → オーガニックの「コスト改善」に吸収
- ❌ 「組織・ガバナンス」を③に書かない → エグゼクティブサマリーの **Open Questions に集約**

各打ち手について **「打ち手 / 仮説の根拠 / 定量インパクト試算（仮置） / 実行リスク」** の4点セットで記述。

#### 3-1. オーガニック

##### (a) 売上

下記4項目を**順番通り**に必ず立てる：

1. **プライシング改善（単純値上げ）** — 価格改定の余地、過去の値上げ実績、転嫁可能性
2. **新規顧客拡大** — 既存セグメント内での新規顧客獲得、商流多角化
3. **新規製品拡充** — 既存技術・チャネルを活用した製品ラインナップ拡大（隣接領域・新規事業のM&A以外の打ち手はここに統合）
4. **新規地域展開** — 既存製品の地理的展開（米欧亜の未開拓地域）

##### (b) コスト

下記2項目を**順番通り**に必ず立てる：

1. **原価改善** — 標準化、オペレーション改善、自動化、調達最適化、生産拠点最適化（旧「オペレーション改善」はここに統合）
2. **販管費改善** — 販売・物流・マーケ・本社機能の効率化、デジタル化、組織スリム化

#### 3-2. インオーガニック

下記2項目を立てる：

1. **同業ロールアップ** — 類似事業を持つ国内外プレイヤーの統合（規模の経済、シェア集約）
2. **隣接企業買収による機能強化** — 技術・顧客・チャネル・地域のいずれかを補完する買収（旧「隣接領域・新規事業」のM&A打ち手はここに統合）

各M&A打ち手について、**候補企業リスト（仮説）、シナジー類型、PMI論点**を含める。

#### 3-3. Exitの可能性（エクイティストーリー含む）

1. **想定買い手** — 事業会社（戦略買収）/ PE（再売却・MBO）/ IPO（再上場・再IPO）の3類型ごとに具体名を仮置で挙げる
2. **想定マルチプル** — 業界EV/EBITDA、PERのレンジ、ピア企業の足元マルチプル
3. **5年後Exit想定** — バリューアップ実行後のEBITDA × Exit時マルチプルで想定EVを試算
4. **エクイティストーリー** — Exit時に買い手・投資家に語るストーリーライン（例：「日本発のグローバルニッチトップとしての成長」「成熟事業から成長事業への構造転換」「アジア展開のプラットフォーム」等）

#### 定量インパクト試算の運用
- 各打ち手につき **売上影響 / EBITDA影響 / 達成期間（年）** をレンジで提示
- 算定根拠（前提となる単価上昇率、シェア獲得幅、コスト削減率等）を必ず併記
- **数字はすべて「仮置」と明示**し、追加DDで詰める前提で提示

#### Value-up Bridge 数値テーブル（③末尾に配置・必須3行構成）

⚠️ **必ず以下の4行（合計を含む）の表で出力する。隣接領域・オペ改善・M&Aを独立行にして5行以上にしてはいけない**：

| 項目 | EBITDA（億円）／レンジ | 寄与の論拠 | 達成期間 |
|---|---|---|---|
| 現状EBITDA | X | 直近実績 | — |
| ＋ オーガニック | +A〜B | 売上ドライバー4項目＋コストドライバー2項目の合算 | 〜N年 |
| ＋ インオーガニック | +C〜D | ロールアップ＋隣接買収の合算 | 〜N年 |
| **5年後想定EBITDA** | **Y〜Z** | — | — |

**個別打ち手まで分解しない**（仮置数値の積み重ねで誤差が大きくなるため）。3-3 Exitの想定EVはBridgeとは別表で示す。

- 出典一覧（有報の頁番号、URL、業界統計の出所、アップロードファイル名）
- 略語集

## 品質基準（厳守）

### 事実と推論の分離
- **確認済事実**には出典を必ず付記
  - 例：[出典: 有報2024年3月期 p.42]
  - 例：[出典: 日経ビジネス2024年X月X日号]
  - 例：[出典: アップロードファイル "○○_業界レポート.pdf" p.15]
- **推論・仮説**は明示的に「（仮説）」「（推定）」「（業界類推）」と書く
- 不明点は「**不明（追加DDで確認要）**」と明記。**推測で空欄を埋めない**
- **取得不可な数値は「不開示」と明記**（仮置による推定値で埋めない）
- **評価・判定（強い/中立/脆弱、A/B/C等）はSkillでは付与しない**

### 数値の取扱い
- すべての数値に出典と時点を付記
- 市場規模・成長率は **計測年（YYYY）** と **期間（始点→終点）** と **実績/予測の別** を明示
- レンジ提示時は前提明記
- 為替・単位を統一

### 文体（読みやすさ重視）

#### 段落分けルール（厳守）

⚠️ **「箇条書きを避けて地の文で書く」=「改行なしの長文ブロックで書く」ではない**

- **段落は2-4文で改行する**：1段落が5文以上になる場合は、論点ごとに段落を分ける
- **異なる論点は別段落**：「数量ドライバーは...」「価格ドライバーは...」「将来見通しは...」等の複数論点を一つの段落に詰め込まない
- **論点の切れ目で必ず改行**：(a)(b)(c)等の小項目は段落を分ける、または箇条書きにする

❌ **悪い例（地の文と改行なし長文を混同）**：

> 数量ドライバーは(a)ミラーレス世代交代に伴う買替需要、(b)動画クリエイター市場の拡大、(c)新興国の所得水準向上による中位機種普及。価格ドライバーはミラーレス対応大口径レンズの単価上昇（一眼レフ用ズーム3-5万円→ミラーレス対応F1.4-2.8単焦点・大口径ズーム10-20万円帯）。市場規模（金額）は本数横ばいでも単価上昇で拡大余地あり（仮説）。

→ 1段落に **3つの論点（数量/価格/全体）** を詰め込み、(a)(b)(c)も段落内に並列配置。読み手が論点の切れ目を追いにくい。

✅ **良い例（論点ごとに段落分け）**：

> 数量ドライバーは3つに分けられる。第1にミラーレス世代交代に伴う買替需要、第2にクリエイター市場の拡大、第3に新興国の所得水準向上に伴う普及だ。
>
> 一方、価格ドライバーはミラーレス対応大口径レンズへの単価上昇シフトである。一眼レフ用ズームの3-5万円帯から、F1.4-2.8単焦点・大口径ズームの10-20万円帯への構造変化が進む。
>
> 市場規模（金額ベース）は、本数が横ばいでも単価上昇により+4-6%程度の拡大が見込まれる（仮説）。

→ 数量・価格・全体の3論点をそれぞれ独立段落に分け、各段落2-3文に収めている。

#### その他の文体ルール
- 各小節の冒頭は**観察事実の要約文（1文）**で始める（仮説の場合「（仮説）」マーカー）
- 続いて短い地の文で論じる（上記の段落分けルール厳守）
- **箇条書きは「列挙が本質的に意味を持つ場面」のみ**（顧客リスト、競合一覧、リスク要因列挙等）
- 数値・比較・推移は**表で示す**

## 過去出力からの教訓（バージョン履歴）

過去のSkill出力で観察された問題と対応：

### v2レポート（タムロン初版）の問題
1. **セグメント別の過去5期の数値が「（仮置）」で埋まっていた**
   → 「過去5期分の有報を個別取得し、各期のセグメント情報注記から数値抽出」を必須化
2. **市場規模の「過去5年」がいつ起点か不明だった**
   → 市場規模テーブルに「期間（始点→終点）」「実績/予測」を必須列化
3. **コスト構造で「材料費等（不明）」と一行で済まされていた**
   → 「製品分解からの素材構成推定」を必須要件化（主要素材2-4個＋市況動向）
4. **表紙・目次で2ページ消費していた**
   → 表紙・目次を廃止し、1行ヘッダから即エグゼクティブサマリー

### v3レポート（タムロン2版）の問題
5. **③バリューアップが旧6章構成（オーガニック/隣接領域/M&A/オペ改善/組織・ガバナンス/Exit）のまま出力された**
   → ⚠️ 構造ガード（禁止事項）を③冒頭に明示し、「3-1オーガニック/3-2インオーガニック/3-3 Exit」の3章構成を強制
6. **オーガニックが「打ち手①②」と通し番号で並び、売上4項目/コスト2項目の構造が無視された**
   → ⚠️ オーガニック直下に通し番号を禁止し、(a)売上4項目+(b)コスト2項目の固定構成を明記
7. **Value-up Bridgeが5行構成（オーガニック/隣接領域/M&A/オペ改善/合計）になった**
   → ⚠️ 必ず3行+合計の4行表で固定する旨を明示
8. **段落が5文以上の長文ブロックになり、論点が3つ以上詰め込まれて読みにくかった**
   → 段落分けルール（2-4文/段落、論点ごとに改行）と悪い例/良い例をSkillに明記
9. **セグメント別5期推移テーブルに構成比列を含めただけで、独立した構成比テーブルがなかった**
   → 必須テーブルAとBを別物として規定し、Bは独立小型テーブルとして必ず作成する旨を明記
