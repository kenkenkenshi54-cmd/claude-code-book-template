# Market Sizing Skill

## このスキルの目的

PE/M&A/コンサルティング実務で必要な市場規模推定を、**説明性・検証性**を重視した方法論で実施する。指定共通フォーマットのエクセルで出力し、IC資料・DDレポートに添付できる体裁とする。

最も重要な原則:
- **複雑な計算は説明性を損なう** → フェルミ推定は3-5要素まで、各因子に出典必須
- **単一推定は脆い** → 三角測量で3独立推定を並べ、収束を確認することが信頼度の根拠
- **過去と将来は連続しない場合がある** → CAGR乖離があれば必ず因子で説明する

## 5ステップ・ワークフロー

### Step 1: 市場定義の明確化

推定する前に、以下を必ず明確化する。

**定義すべき5項目**:
1. **製品・サービスの範囲**: 何を含み、何を含まないか
2. **地理的範囲**: 日本国内 / アジア / グローバル
3. **バリューチェーン上の位置**: 完成品 / 部品 / 原材料
4. **顧客セグメント**: B2B / B2C, OEM / アフターマーケット
5. **時点**: 直近実績年度 / 中期見通し

**良い定義の例**: 「2024年度 日本国内 OEM向け乗用車内装表皮材市場(本革・合皮・ファブリック含む、計装材・シートクッション材を除く)」

ユーザーの初期インプットが曖昧な場合は、推定を始める前に**必ず定義を確認する**。

### Step 2: 公式統計の優先確認 (推定①の主要候補)

フェルミ推定に進む前に、必ず公式統計を探す。統計が見つかれば、それが**推定①**となる。

**日本国内市場 (優先順)**:
1. **業界団体統計**: 自工会、日本鉄鋼連盟、日本電機工業会、日本化学工業協会、SEAJ、JCMA等
2. **経済産業省統計**: 工業統計、生産動態統計、商業動態統計、企業活動基本調査
3. **政府白書・年次報告**: ものづくり白書、自動車白書、エネルギー白書等
4. **民間調査会社レポート**: 矢野経済、富士経済、IDC Japan、富士キメラ総研等

**グローバル市場**: グローバル業界団体 (OICA、SEMI、WSTS等)、調査会社 (S&P/IHS、Omdia、IDC、Gartner、MBB Industry Reports)

統計確認時の注意点:
- **定義範囲が一致しているか必ず確認**(調査機関ごとにスコープが違う)
- **集計年次** / **集計方法** (出荷額 / 生産額 / 販売額) を確認

### Step 3: フェルミ推定 (公式統計が無い場合の推定①)

**鉄則**: 計算式は3-5要素まで。説明性が最優先。

**代表的なアプローチ**:

| アプローチ | 計算式の型 | 例 |
|---|---|---|
| 完成品市場 | 国内出荷台数 × 平均単価 | 日本の乗用車市場 = 国内販売台数 × 平均車両価格 |
| 部品市場 | 親市場 × 部品単価比率 × 装着率 | 内装表皮材 = 国内生産台数 × 1台当たり使用量 × 単価 |
| サービス市場 | 顧客数 × 顧客単価 × 利用頻度 | カーシェア市場 = 会員数 × 利用額 × 利用回数 |
| 設備投資型市場 | 設置ベース × 更新サイクル × 単価 | 工作機械 = 稼働台数 / 更新年数 × 平均単価 |

**避けるべきパターン**:
- ✗ 出典不明の数字を組み合わせる(感覚値)
- ✗ 細分化しすぎて検証不能(車種別×グレード別×素材別×地域別…)

各因子の値の隣セル(U列・V列)に出典・URLを必ず記載する。

### Step 4: 三角測量 (3独立推定の収束検証)

**核心**: Step 2/3 で得た推定①は、それ単独では信頼できない。**異なる方法論で得た推定②・③と並べて、収束しているかを確認**することが本質。

#### 推定② シェア × 売上 から逆算

業界紙・リサーチレポートで判明する「シェア × 売上」のペアから市場規模を逆算。

例:
- A社のシェア = 30%、A社の当該セグメント売上 = 410億円
- 逆算市場規模 = 410 / 0.30 ≈ 1,367億円

複数企業で実施しレンジを把握。シェアの定義(数量/金額、国内/グローバル)が統一されているか要確認。

#### 推定③ 主要プレーヤー合計 / カバー率

主要プレーヤー(上位5-10社)の当該セグメント売上を以下から拾う:
- 有価証券報告書のセグメント情報
- 決算説明資料・IR資料
- 中期経営計画資料
- 業界紙報道(日経産業新聞、業界専門紙)

逆算市場規模 = プレーヤー売上合計 ÷ 想定カバー率
- 寡占市場: カバー率 70-90%
- 中小プレーヤー乱立: 30-50%

例: 上位4社の合計が 1,095億円、カバー率 80% → 1,095 / 0.80 ≈ 1,369億円

#### 採用値の決定

3推定 (①②③) が概ね収束している場合(差異 ±10%程度以内):
- 中央値 / 主要手法 (通常①) を採用値とする
- テンプレート Row 7 K列 = 採用した推定行への formula link (例: `=K10`)

3推定が大きく乖離している場合:
- どの推定の前提に問題があるか検討
- 市場定義が一致していない、シェアの分母定義が違う、特定企業のセグメント区分が違う等
- 必要に応じて Step 1 (市場定義) に戻る

#### 収束評価の目安

| 3推定の幅 | 評価 | アクション |
|---|---|---|
| ±5% 以内 | 強く収束 | 推定値の信頼度 High |
| ±5〜15% | 概ね収束 | 中央値採用、レンジで提示 |
| ±15〜30% | やや乖離 | 各推定の前提を再確認、レンジを広めに提示 |
| ±30% 超 | 大きく乖離 | 市場定義に立ち戻る。単一推定の採用は避ける |

### Step 5: 将来CAGRの妥当性チェック (過去CAGRとの乖離分析)

市場規模を年次で試算した場合、**過去実績期間のCAGR と 将来計画期間のCAGR を必ず比較する**。乖離がある場合は、その差分を構成する因子を**追加された成長要因**と**消滅・減少した要因**に分けて明示する。

これは「ホッケースティック型の根拠なき成長前提」を排除するための検証であり、IC・DDで最も突かれるポイント。

#### 手順

1. **乖離の数値化** (テンプレート Row 36-38)
   - 過去CAGR と 将来CAGR を Row 7 から リンク (`=S7`, `=T7`)
   - 乖離 (将来 − 過去) を計算 (単位: pp = percentage point)

2. **因子の分解** (テンプレート Row 41-44)
   - **(+) 新たに加わった成長ドライバー**: 過去期間に存在せず、計画期間で寄与する要因
     - 例: 規制変更による新規需要、技術トレンド転換 (EV化、AI実装、CPO化等)、新規顧客セグメント開拓、海外展開、価格改定の浸透
   - **(-) 消滅・減少した要因**: 過去期間に成長を支えていたが、計画期間では弱まる/消える要因
     - 例: 構造的需要縮小 (人口減・国内市場縮小)、補助金終了、競争激化による単価下落、技術代替による既存製品陳腐化

3. **定量化** (テンプレート K41-K44)
   - 各因子の影響を**pp(percentage point)単位で粗く**見積もる
   - 厳密な算定は不要。**主要因子の寄与順位と概算スケール感**が示せれば十分

4. **整合性チェック** (テンプレート Row 46-47)
   - 要因合計と乖離の残差 (K47) を確認
   - 残差が大きい (例: |残差| > 0.5pp) 場合、要因の網羅性が不足

#### 評価コメントの目安

| 乖離 | 解釈 | アクション |
|---|---|---|
| ±0.5pp 以内 | 過去トレンド継続前提 | 形式的でOK |
| 過去より +1〜+3pp | 加速ストーリー | 加速ドライバーを定量必須 |
| 過去より +3pp 超 | 不連続な成長前提 | 構造変化を明示。**根拠が薄い場合は推定値の下方修正検討** |
| 過去より -1pp 以下 | 減速ストーリー | 減速要因を定量必須 |

## エクセル出力フォーマット (指定テンプレート)

### テンプレート使用

`assets/template.xlsx` を**必ずベース**として使用する。コピー後、市場ごとにシートを作成・記入する。

```python
import shutil
```

### 列構造 (全シート共通・固定)

| 列 | 用途 |

### 色分け規則

- **青字 RGB(0,112,192) = `0070C0`**: ベタ打ち入力値、感応度を変えたい数値
- **黒字 RGB(0,0,0)**: 計算式・ラベル
- **緑字 RGB(0,128,0)**: 他シート参照(複数市場統合時)

```python
from openpyxl.styles import Font
blue_font = Font(color='0070C0')   # ベタ打ち
black_font = Font(color='000000')  # 計算式
green_font = Font(color='008000')  # クロスシート参照
```

### 数値書式

- 億円・百万円: `#,##0;(#,##0);"-"`
- パーセント: `0.0%`
- 倍率: `0.0x`
- 年: テキスト扱い (例: `"2024"`)

## 複数市場の取り扱い

### シート分割ルール

- **1市場 = 1シート**
- シート名 = 市場名 (例: 「内装表皮材_日本」「車載半導体_グローバル」)
- すべてのシートで**列構造・行構造を完全に共通化**する

### 後段の統合性を担保するため

- 年次列の位置 (H〜Q) は全シートで固定
- 単位は揃える (億円なら全シート億円)
- 主要 KPI 行 (Row 7=市場規模、Row 13=三角測量採用値) も共通行番号に置く
- これにより `=SUM('日本'!K7,'米国'!K7,'欧州'!K7)` のようなクロスシート集計が可能

### シート追加時のコード例

```python
new_sheet = wb.copy_worksheet(wb['A市場'])
new_sheet.title = '車載半導体_グローバル'
# 値を上書き・更新
```

## ユーザーが情報を提供しない場合の進め方

1. まず公式統計を web_search で探す → 推定①
2. 統計が見つからなければフェルミ推定 → 推定①
3. 検算用に競合企業のシェア・売上を web_search → 推定②③
4. 不足分はユーザーに具体的に依頼: 「[A社] の[セグメント]売上が確認できれば収束を検証できます」

無闇に推定値を出さず、**「この情報があれば信頼度を上げられる」**を明示することを優先する。
